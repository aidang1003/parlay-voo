# Frontend (Next.js 14 / wagmi / viem / ConnectKit)

## Stack

Next.js 14 (App Router), React 18, TypeScript, Tailwind 3. Wallet: wagmi 2, viem 2, ConnectKit. AI: Vercel AI SDK + Claude. Deployed to Vercel.

## Pages

- `/` -- parlay builder
- `/vault` -- LP dashboard
- `/tickets` -- user tickets list
- `/ticket/[id]` -- ticket detail + settle/claim
- `/about` -- project overview

## API Routes

- `/api/chat` -- AI chat (streaming, tool calling)
- `/api/mcp` -- MCP JSON-RPC endpoint
- `/api/markets` -- Market catalog
- `/api/quote-sign` -- EIP-712 signed JIT quotes
- `/api/premium/agent-quote` -- Agent quote + risk

## Key Files

- `contracts/deployedContracts.ts` -- AUTO-GENERATED by `scripts/generate-deployed-contracts.ts`. Source of truth for addresses + ABIs. Do not edit by hand.
- `hooks/useDeployedContract.ts` -- wagmi hook for contract lookup by chain. Falls back to first available chain.
- `lib/contracts.ts` -- backwards-compat shim re-exporting from deployedContracts.ts
- `lib/hooks.ts` -- wagmi hooks (`isPending -> isConfirming -> isSuccess`)
- `lib/wagmi.ts` -- wagmi config (foundry + baseSepolia), RPC URLs from `@parlaycity/shared` chains.ts
- `lib/mcp/tools.ts` -- 6 MCP tool implementations

## Rules

- Never hardcode contract addresses. They come from `deployedContracts.ts` (generated from forge broadcast).
- Protocol constants (fees, limits) import from `@parlaycity/shared` constants.ts -- no local config file.
- Chain config (RPC URLs, chain IDs) imports from `@parlaycity/shared` chains.ts.
- USDC is 6 decimals: `parseUnits("100", 6)`.
- Polling: 5s for tickets/balances, 10s for vault stats.
- Each transaction button gets its own loading state.

## Hot Reload on Redeploy

`next.config.mjs` sets webpack `managedPaths: []` + `followSymlinks: true` (SE2 pattern) so that regenerating `deployedContracts.ts` triggers Fast Refresh without restarting the dev server.

## Testing

`pnpm test` runs vitest. `npx tsc --noEmit` for typecheck.
